# 暗色主题实现计划

## 背景

LiteMart POS 当前没有任何暗色模式基础设施。项目使用 Tailwind CSS 3.4，样式分两种模式：

- **10 个页面**：纯 Tailwind 工具类（slate/zinc 色系）
- **2 个页面**（login.vue、members.vue）：scoped CSS + 硬编码十六进制颜色值
- **dashboard.vue**：SVG 内联 RGB 颜色值

本方案采用 Tailwind 的 `darkMode: 'class'` 策略，通过在 `<html>` 元素切换 `.dark` 类来实现暗色主题。

## 第二阶段：主题状态管理 + 防闪烁

### 2.1 新建 `app/stores/useTheme.ts`

Pinia Store，管理主题偏好：

- **state**: `mode: 'light' | 'dark' | 'system'`（默认 `system`）
- **getter**: `resolved` — 根据 mode 和系统偏好返回实际的 `light` 或 `dark`
- **actions**:
  - `setMode(mode)` — 切换主题并应用
  - `applyTheme()` — 在 `<html>` 上切换 `.dark` 类
  - `initSystemListener()` — 监听系统主题变化
- **持久化**: 使用 `pinia-plugin-persistedstate` 保存到 localStorage

### 2.2 新建 `app/plugins/theme.client.ts`

客户端插件，在 Vue 水合之前同步读取 localStorage，立即设置 `.dark` 类，**防止页面加载时闪烁**。

### 2.3 更新 `app/app.vue`

- 使用 `useHead()` 同步 `<html>` 的 class 和 `<meta name="color-scheme">`
- `onMounted` 时调用 `applyTheme()` 和 `initSystemListener()`

## 第五阶段：login.vue + members.vue 的 scoped CSS 改造

将这两个页面中所有硬编码的十六进制颜色值替换为第一阶段定义的 CSS 变量。

**login.vue 改造示例：**

```css
/* 改造前 */
.form-panel {
  background: #fafaf9;
}
.heading-title {
  color: #1c1917;
}
.btn-primary {
  background: #1c1917;
  color: #fafaf9;
}
.field-input {
  background: white;
  color: #1c1917;
}
.msg-error {
  background: #fff1f2;
  border-color: #fecdd3;
  color: #be123c;
}

/* 改造后 */
.form-panel {
  background: var(--bg-page);
}
.heading-title {
  color: var(--text-primary);
}
.btn-primary {
  background: var(--btn-primary-bg);
  color: var(--btn-primary-text);
}
.field-input {
  background: var(--bg-input);
  color: var(--text-primary);
}
.msg-error {
  background: var(--bg-error);
  border-color: var(--border-error);
  color: var(--color-error);
}
```

**members.vue 特殊处理：**

- `LEVEL_CONFIG` 中的 `NORMAL` 等级使用 CSS 变量，`SILVER`/`GOLD` 保持原色（亮色下已经足够醒目）
- 滚动条颜色 `#d6d3d1` → `var(--scrollbar-thumb)`
- 下拉箭头 SVG data URI 中的颜色可保持不变（两种模式下都可接受）

## 第六阶段：Dashboard SVG 颜色

将 dashboard.vue 中的硬编码 SVG 颜色替换为 CSS 变量：

```html
<!-- 改造前 -->
<line stroke="rgb(226 232 240)" ... />
<path stroke="rgb(79 70 229)" ... />
<circle stroke="rgb(16 185 129)" ... />

<!-- 改造后 -->
<line stroke="var(--chart-line-light)" ... />
<path stroke="var(--chart-line)" ... />
<circle stroke="var(--chart-emerald)" ... />
```

对于 `<linearGradient>` 的 `<stop>` 元素，使用计算属性绑定 `:stop-color` 以确保兼容性。
