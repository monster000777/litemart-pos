# 注册登录功能完善 — 进展文档

> 更新时间：2026-04-27 09:15

## 已完成改动

### 1. 登录页功能逻辑增强 (`app/pages/login.vue`)

- **PIN 重置流程**：登录页底部「重置 PIN」入口，3步流程：旧PIN → 新PIN → 确认新PIN
- **物理键盘支持**：数字键输入 + Backspace 删除 + Enter 提交
- **锁定倒计时 UI**：被速率限制时显示黄色倒计时提示，禁用键盘
- **消息互斥展示**：锁定/错误/帮助消息使用 v-else-if 链，避免同时显示
- **Session 过期回跳**：登录后自动跳回之前访问的页面
- **UI 保持原始白色简洁风格**

### 2. PIN 安全增强

#### 速率限制器 (`server/utils/rate-limiter.ts`)

- 内存级速率限制器，全局单例模式
- 5分钟窗口内 5 次失败锁定 60 秒
- 每 10 分钟自动清理过期条目防内存泄漏
- 登录成功自动重置计数

#### 登录 API (`server/api/auth/login.post.ts`)

- 集成速率限制，PIN 验证前检查是否被锁定
- 失败时记录尝试次数，触发锁定时返回锁定秒数

#### 重置 PIN API (`server/api/auth/reset-pin.post.ts`)

- 旧 PIN 验证也受速率限制保护，防暴力破解

### 3. PIN 重置功能

#### 重置 API (`server/api/auth/reset-pin.post.ts`)

- 提交旧 PIN + 新 PIN + 确认 PIN
- 服务端验证：旧PIN正确 → 新PIN != 旧PIN → 新PIN格式一致 → 更新数据库
- 重置后自动签发新 session

#### auth-config-service 扩展

- 新增 `updateAuthConfig(hashedPin)` 方法

### 4. Session 管理优化 (`app/middleware/auth.global.ts`)

- 过期时保存当前路由作为 `redirect` 参数
- 登录成功后回跳原页面

### 5. 项目结构优化

- `server/utils/request.ts` — 提取 `getClientIp` 共享工具函数，消除重复代码
- `server/utils/rate-limiter.ts` — 速率限制器独立模块
- 服务端中间件白名单已包含 `reset-pin` 路径

## Bug 修复记录

1. **消息同时显示** — login.vue 中锁定提示和错误消息可能同时渲染，改为 `v-else-if` 互斥链
2. **resetPin 冗余赋值** — catch 中 `resetStep.value = STEP_NEW_CONFIRM` 无效赋值已移除
3. **reset-pin 无速率限制** — 旧PIN验证未受保护，已加入速率限制防暴力破解
4. **getClientIp 重复定义** — 两个 API 文件各自内联定义，提取为 `server/utils/request.ts` 共享
