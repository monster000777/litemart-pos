## 二、数据库设计

### 2.1 新增 Customer 表

```prisma
model Customer {
  id        String   @id @default(cuid())
  phone     String   @unique
  name      String?
  points    Int      @default(0)
  level     String   @default("NORMAL")
  createdAt DateTime @default(now())
  orders    Order[]

  @@index([phone])
  @@index([level])
}
```

**字段说明：**

- `phone`：手机号，唯一标识
- `name`：会员姓名（可选）
- `points`：当前积分余额
- `level`：会员等级（NORMAL/SILVER/GOLD）

### 2.2 修改 Order 表

```prisma
model Order {
  // ... 现有字段
  memberId String?
  member   Customer? @relation(fields: [memberId], references: [id])
}
```

### 2.3 新增 PointLog 表（积分变动记录）

```prisma
model PointLog {
  id         String   @id @default(cuid())
  customerId String
  customer   Customer @relation(fields: [customerId], references: [id])
  orderId    String?
  change     Int      // 正数为增加，负数为扣减
  reason     String   // 如：消费积分、积分抵扣、手动调整
  createdAt  DateTime @default(now())

  @@index([customerId])
  @@index([orderId])
}
```

### 2.4 Product 表扩展（会员价）

```prisma
model Product {
  // ... 现有字段
  memberPrice Decimal?  // 会员价（可选）
}
```

## 五、业务逻辑

### 5.1 积分规则

| 场景     | 规则              | 说明                 |
| -------- | ----------------- | -------------------- |
| 消费积分 | 1元 = 1积分       | 订单完成后自动累加   |
| 积分抵扣 | 100积分 = 1元     | 结算时可选择抵扣金额 |
| 最低抵扣 | 100积分起         | 不足100积分不可抵扣  |
| 最高抵扣 | 不超过订单金额50% | 防止零元订单         |

### 5.2 会员价规则

- 商品可设置专属会员价
- 会员价仅对会员生效
- 会员价与积分抵扣可叠加使用
- 优先显示会员价，再计算积分抵扣

### 5.3 会员等级

| 等级   | 条件             | 权益      |
| ------ | ---------------- | --------- |
| NORMAL | 注册即得         | 基础积分  |
| SILVER | 累计消费满1000元 | 积分1.2倍 |
| GOLD   | 累计消费满5000元 | 积分1.5倍 |
